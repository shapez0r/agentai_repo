# Budlendar Deployment Workflow

## Standard Workflow for All Tasks

After completing any code changes or feature additions:

### 1. Git Commit & Push
```bash
git add -A
git commit -m "<descriptive message>"
git push
```

### 2. Build & Deploy
```bash
npm run build
npm run start:local
```

### 2a. Hidden Deployment Requirement
- **Do not open `cmd`, PowerShell, Terminal, or any separate console window** for deployment or verification
- **Deployment must stay internal to the AI VS Code plugin workflow**
- **Use only the hidden local deployment entrypoint**: `npm run start:local`
- `npm run start:local` must run the production-style app in the background with no visible console popups
- If a deployment approach would open an external shell window, **do not use it**; fix the workflow to stay hidden instead

### 3. Verify App Actually Runs
**CRITICAL**: After deploying, ALWAYS verify the app actually loads and works:
- Check that the server starts without errors
- Verify the UI loads in the browser
- Test that API calls work (e.g., try to log in)
- If the app doesn't run or shows errors, FIX the issue immediately
- Do NOT mark task complete until the app successfully runs
- Once fixed, git commit, push, and rebuild before completion

### Quick Reference
- **Always commit and push changes** before redeploying
- **Always rebuild** (`npm run build`) after code changes
- **Always redeploy** with the hidden internal workflow to verify changes work in production-like environment
- **ALWAYS verify the app actually runs** - if it fails to load, fix it and redeploy
- **Never launch visible consoles** during deploy or verification; everything must stay inside the AI VS Code plugin
- **Never skip the deploy step** - verification is part of task completion

## Port Configuration

**IMPORTANT**: Always keep both dev and prod servers running for testing!

### Development Mode (npm run dev)
- **Client**: http://127.0.0.1:8785
- **API**: http://127.0.0.1:8786
- Vite proxies `/api` requests from 8785 to 8786

### Production Mode (npm run start:local)
- **Client**: http://127.0.0.1:8787
- **API**: http://127.0.0.1:8788
- Deployment must run hidden with no visible console windows
- The AI VS Code plugin should invoke this internally rather than opening `cmd` or PowerShell

### Configuration Files
- `vite.config.js`: Dev client port 8785, prod preview port 8787, proxies to appropriate API
- `server/dev.js`: Sets API_PORT to 8786 and APP_ORIGIN to dev client (8785)
- `server/config.js`: Default API_PORT is 8787 (used by production)

### Other Settings
- **Database**: SQLite (local, at `server/data/` when running)
- **Auth**: Local implementation with session tokens in HTTP-only cookies
- **Build Output**: `dist/` directory

## Known Issues & Fixes

### Issue: App doesn't load in dev mode
**Cause**: The vite.config.js proxy was incorrectly pointing to 8787 instead of 8788
**Fix**: Ensure `vite.config.js` has proxy target set to `http://127.0.0.1:8788`
**Verification**: When running `npm run dev`, the Vite client should successfully connect to the API server without proxy errors

## When Making Changes
1. Make code changes
2. Test locally in dev mode if needed
3. Stage changes: `git add -A`
4. Commit with descriptive message
5. Push to origin
6. Build: `npm run build`
7. Deploy: `npm run start:local` using the hidden internal plugin workflow only
8. Confirm task is complete
