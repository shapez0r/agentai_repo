# Budlendar Deployment Workflow

## Standard Workflow for All Tasks

After completing any code changes or feature additions:

### 1. Git Commit & Push
```bash
git add -A
git commit -m "<descriptive message>"
git push
```

### 2. Build & Deploy
```bash
npm run build
npm run start:local
```

### 3. Verify App Actually Runs
**CRITICAL**: After deploying, ALWAYS verify the app actually loads and works:
- Check that the server starts without errors
- Verify the UI loads in the browser
- Test that API calls work (e.g., try to log in)
- If the app doesn't run or shows errors, FIX the issue immediately
- Do NOT mark task complete until the app successfully runs
- Once fixed, git commit, push, and rebuild before completion

### Quick Reference
- **Always commit and push changes** before redeploying
- **Always rebuild** (`npm run build`) after code changes
- **Always redeploy** to verify changes work in production-like environment
- **ALWAYS verify the app actually runs** - if it fails to load, fix it and redeploy
- **Never skip the deploy step** - verification is part of task completion

## Project Configuration
- **Port**: 8787 (main app port - do NOT change)
- **Dev Mode**: 
  - Vite client: port 8787
  - Express API: port 8788
  - Vite proxies `/api` requests to 8788
- **Production Mode**: Both client and API on port 8787 (Express serves static files)
- **Database**: SQLite (local, at `server/data/` when running)
- **Auth**: Local implementation with session tokens in HTTP-only cookies
- **Build Output**: `dist/` directory

## Known Issues & Fixes

### Issue: App doesn't load in dev mode
**Cause**: The vite.config.js proxy was incorrectly pointing to 8787 instead of 8788
**Fix**: Ensure `vite.config.js` has proxy target set to `http://127.0.0.1:8788`
**Verification**: When running `npm run dev`, the Vite client should successfully connect to the API server without proxy errors

## When Making Changes
1. Make code changes
2. Test locally in dev mode if needed
3. Stage changes: `git add -A`
4. Commit with descriptive message
5. Push to origin
6. Build: `npm run build`
7. Deploy: `npm run start:local` (or verify the server starts correctly)
8. Confirm task is complete
